# 기기별 개인 하이라이트 (Personal Highlights) — 설계

- 날짜: 2026-07-01
- 대상: 시간표 PWA (`C:\Users\wbnuj\timetable-pwa`, 배포 `pureart-art/Timetable26-1`)
- 상태: 승인됨 (구현 대기)

## 배경 / 문제

시간표는 반 공용 자료다(친구들도 각자 웹 앱으로 본다). 지금은 색이 **구글 시트 셀 서식**에서 나온다: 교수=검정, 초안자/검안자=파랑, 시험=빨강. 모두가 같은 시트를 읽으므로, GY가 자기 이름·지도/동문 교수를 시트에서 빨갛게 하면 **모두가** 그 빨강을 본다.

문제 두 가지:
1. 친구들이 GY의 개인 표시(빨강)를 강제로 보게 됨 — 공용 자료에 한 사람 관점이 얹힘.
2. 빨강이 "시험(공용 신호)"과 "GY 개인 표시" 두 의미로 섞여 **가장 중요한 공용 신호(시험)가 흐려짐**.

## 목표

- **누구나** 자기 기기에서 자기 이름/키워드를 넣어 해당 줄을 빨강으로 볼 수 있는 **범용 개인화 기능**. (GY만이 아니라 친구들도 각자 사용 — 각자 localStorage라 서로 독립.)
- GY의 경우: 자기 이름이 든 초안자/검안자 줄 + 자기 지도/동문 교수 줄을 빨강으로.
- **키워드를 안 넣은 사람이 열면** 모든 항목이 중립색(교수 검정·초안자 파랑)으로 — 즉 현행과 동일.
- **시험 빨강**은 어디서 열든 모두에게 그대로 유지된다.
- 로그인/서버 없이, 순수 클라이언트 사이드로 구현한다.
- **[최우선 제약] 현재 앱·위젯을 쓰는 친구들에게 무오류·무변화.** 개인화는 순수 additive: 키워드가 비면(친구) 렌더 결과가 **현행과 픽셀 동일**해야 한다. 구현·검증의 1순위 합격 기준.

## 비목표 (YAGNI)

- 사용자 신원/로그인 기반 개인화.
- 사람마다 시트·배포를 따로 두기.
- 카테고리별(내 이름 vs 내 교수) 분리 색상 — 지금은 하나의 평평한 키워드 목록으로 충분.
- 매칭된 이름 글자만 부분 색칠 — 줄 전체를 칠한다(요청: "섹션 빨강").

## 설계 결정 (확정)

- **활성화 방식**: 앱 내 설정 패널 + localStorage (기기별 영구).
- **적용 범위**: 웹 앱 + Scriptable 위젯 둘 다.
- **개인색 처리**: 시험과 **같은 빨강** 그대로. 시험=제목줄, 개인=초안자/교수줄이라 위치로 구분.
- **설정 진입점**: 버전/갱신 기록 옆 **작게 보이는 톱니(⚙) 버튼** → 모달. (누구나 발견·사용 가능. 롱프레스 방식 폐기.)
- **모달 안내 문구**: "본인 이름/키워드를 넣으면 그 줄이 *이 기기에서만* 빨갛게 표시됩니다. 다른 사람에겐 영향 없어요."
- **문서화**: 설정 모달 안 + `guide.html` 사용법 섹션 둘 다에 설명 추가.
- **매칭**: 양쪽 **공백 전부 제거 + 소문자화** 후 부분일치. 매칭된 **줄 전체**를 빨강.
  - 학생(초안자/검안자)은 **이름만**(예: 이름 문자열) → 해당 줄 매칭.
  - 교수는 **`(이름, 과)` 통째로** → 같은 이름 다른 과 오탐 배제.
- **시트**: 중립화(개인 수동 빨강 제거, 시험 빨강만 유지) — GY가 구글 시트에서 1회 수행.
- **위젯 키워드 출처**: 공개 레포에 실명 노출을 피하려고, 커밋되는 위젯 스크립트는 **빈 목록**(`HL_KEYWORDS = []`)을 기본값으로 두고, 실제 키워드는 **레포 밖 로컬 Scriptable 파일**(iCloud documents, 예: `tt-hl.txt`)에서 읽는다. GY가 그 파일을 1회 생성.
- **키워드 프라이버시**: GY의 실제 키워드 목록(학생명 1 + 교수 `(이름,과)` 11 ≈ 12개)은 **어디에도 커밋하지 않는다** — 앱은 localStorage, 위젯은 로컬 파일. 이 스펙 문서에도 실제 이름을 적지 않는다(별명↔실명 비연결 유지).

## 아키텍처

개인화는 **렌더 직전 덧칠 레이어**다. `parseGrid`가 만든 `weeks[].cells[].lines[{text,color}]` 위에, 키워드에 걸리는 줄의 색만 빨강으로 바꾼다.

핵심 불변식: **`isExam`은 `parseGrid`에서 시트 원색으로 이미 계산되므로 오버레이가 건드리지 않는다** → 시험 빨강은 개인화와 독립적으로 모두에게 유지된다.

### 데이터 흐름 (앱)

```
fetchLive()/loadSnapshot()
   → parseGrid(json)            # lines[].color = 시트 원색, isExam 확정
   → applyHighlights(weeks, kw) # kw에 매칭되는 line.color = RED (렌더 직전)
   → render()
```

- `applyHighlights`는 `applyData()` 안에서 `render()` 직전에 호출.
- 키워드가 비면(친구) 아무 것도 안 바꿈 → 완전 중립.
- 데이터 갱신(스냅샷/라이브)마다 `parseGrid`가 lines를 새로 만들고 오버레이가 다시 적용되므로 상태가 항상 일관됨.
- 설정 저장 시에도 즉시 재적용 후 `render()`.

### 데이터 흐름 (위젯)

위젯 스크립트 자체는 공개 레포에 있으므로 키워드를 하드코딩하지 않는다. 커밋본은 `HL_KEYWORDS = []`(빈 목록)이고, 실행 시 레포 밖 로컬 파일(`tt-hl.txt`)을 읽어 채운 뒤, 칸 그리기 직전에 앱과 같은 덧칠 로직을 적용. 파일이 없으면 빈 목록 → 현행과 동일.

## 컴포넌트 / 유닛

### 앱 (`app.js`)
- `const HL_KEY = 'tt_hl'` — localStorage 키.
- `getKeywords(): string[]` — localStorage에서 읽어 줄바꿈/쉼표 분리·trim·빈값 제거.
- `saveKeywords(raw: string)` — 저장 후 `applyHighlights` + `render`.
- `matchKeyword(text, keywords): boolean` — 양쪽 `replace(/\s+/g,'').toLowerCase()` 후 부분일치.
- `applyHighlights(weeks, keywords)` — 각 cell의 각 line에 대해 매칭 시 `line.color = HL_RED`.
- 설정 모달: 최소 HTML(기존 `#sheetpop` 스타일 재사용) + 안내 문구 + textarea + 저장/닫기. 진입점 = 헤더 메타행(버전·갱신 옆)에 작은 `⚙` 버튼(클릭). 모바일 폭에서 레이아웃 안 깨지게 배치.
- `const HL_RED = '#FF0000'` (또는 시험 판정 `isRedHex`를 통과하는 값; 시각적으로 시트 시험 빨강과 동일 계열).

### 가이드 (`guide.html`)
- 개인화 사용법 섹션 1개 추가: ⚙ 버튼 위치, 이름 넣는 법, "이 기기에서만 적용·남에게 영향 없음" 설명. (선택) 위젯용 `tt-hl.txt` 안내는 GY용 별도 메모.

### 위젯 (`widget/timetable-widget.js`)
- 커밋 기본값 `let HL_KEYWORDS = [];` (빈 목록 → 하이라이트 없음 → 친구/타인 실행해도 안전, 실명 노출 0).
- 실행 시 **레포 밖 로컬 파일**을 읽어 키워드를 채운다: `FileManager.iCloud()`(또는 `.local()`)의 documents 디렉터리에서 `tt-hl.txt`(한 줄에 하나 또는 쉼표 구분)를 있으면 로드, 없으면 빈 목록 유지. 파일 접근 실패는 조용히 무시(빈 목록).
- 동일 `matchKeyword` + 칸 렌더 직전 매칭 줄 색을 빨강으로.
- GY 액션: Scriptable iCloud 폴더에 `tt-hl.txt` 1회 생성(자기 키워드 붙여넣기). 레포엔 안 올라감.

### 시트 (수동, 1회)
- GY가 구글 시트에서 자기 이름 초안자/검안자 줄을 파랑으로, 자기 교수를 검정으로 되돌림. 시험 빨강은 유지. 다음 30분 스냅샷 크론이 자동 반영.

## 엣지 케이스 / 주의

- **동명이인 오탐**: 교수는 `(이름, 과)` 통째 매칭으로 과가 다른 동명이인 배제. 학생은 이름만 매칭이라 동명이인 학생이 있으면 걸릴 수 있음 — 드물어 감수.
- **개인 빨강 = 시험 빨강(동색)**: 의도된 선택. 위치(제목줄 vs 하위줄)로 구분.
- **매칭 줄이 마침 제목줄(과목명)일 때**: 이미 시험이면 빨강 유지(무해). 시험 아닌 제목줄이 키워드에 걸리면 빨강이 되는데, 보통 제목엔 이름/교수가 없어 실제로는 드묾.
- **앱 localStorage와 위젯 로컬 파일 이중 관리**: 런타임이 달라(브라우저 vs Scriptable) 저장소 공유 불가. 목록이 자주 안 바뀌므로 감수(각 기기 1회).
- **친구도 설정 사용은 의도된 것**: ⚙ 버튼은 공개 기능. 각자 localStorage라 서로 독립 — 한 사람 설정이 남에게 안 보임. 안 넣으면 현행 그대로.
