## 1단계. 구글 시트 공유 설정

1. [시트](https://docs.google.com/spreadsheets/d/1ApcQRzpRt5J3qHysCEq7uzCekYp_RusG/edit) 열기 → 우상단 **공유**
2. **링크가 있는 모든 사용자** → **뷰어** 로 설정

## 2단계. Google Cloud에서 API 키 발급

1. https://console.cloud.google.com 접속(구글 계정 로그인) → 상단 프로젝트 선택 → **새 프로젝트** (이름: `timetable` 등)
2. 메뉴 → **API 및 서비스 → 라이브러리** → "Google Sheets API" 검색 → **사용 설정**
3. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → API 키**
4. 키가 만들어지면 **키 제한**(중요 — 키가 HTML에 노출되므로 꼭):
   - **애플리케이션 제한사항**: 웹사이트 → `https://<깃허브아이디>.github.io/*` 추가 (로컬 테스트도 하려면 `http://localhost:8741/*` 추가)
   - **API 제한사항**: 키 제한 → **Google Sheets API**만 선택
5. 키 복사

> 키가 노출되어도 이 키로 할 수 있는 일 = "공개된 시트 읽기"뿐이라 위험은 낮지만, 제한은 걸어두는 게 좋습니다.

## 4단계. GitHub 저장소 만들기 + 업로드

### 방법 A — 웹 브라우저만으로 (제일 쉬움)

1. https://github.com 가입/로그인 → 우상단 **+** → **New repository**
2. 이름: `timetable` (원하는 이름), **Public**, 그대로 **Create repository**
3. 저장소 화면에서 **uploading an existing file** 클릭 → 탐색기에서
   `index.html, app.js, styles.css, manifest.json, sw.js, icon-192.png, icon-512.png` 드래그 → **Commit changes**
4. `data` 폴더는 웹 업로드에서 폴더째 드래그하면 됩니다 (`data/snapshot.js` 포함되도록).

> 비공개로 하고 싶으면 Private 저장소 + GitHub Pages는 유료(Pro)라서, 동기 몇 명 공유 용도면 **Public 저장소 + 추측 불가 URL**로 충분합니다 (브리프 §10 합의사항).

## 5단계. GitHub Pages 켜기

1. 저장소 → **Settings → Pages**
2. **Source**: Deploy from a branch / **Branch**: `main`, 폴더 `/ (root)` → **Save**
3. 1~2분 후 `https://<깃허브아이디>.github.io/timetable/` 접속 → 시간표가 떠야 함
4. 2단계의 키 제한 리퍼러가 이 주소와 일치하는지 확인

## 6단계. 기기에 설치 (PWA)

| 기기 | 방법 |
|---|---|
| iPhone / iPad | Safari로 접속 → 공유 버튼 → **홈 화면에 추가** |
| Windows | Chrome/Edge로 접속 → 주소창 오른쪽 **설치** 아이콘 (또는 ⋯ 메뉴 → 앱 설치) |
| Mac | Chrome → 주소창 설치 아이콘 / Safari 17+ → 파일 → Dock에 추가 |

설치하면 일반 앱처럼 아이콘으로 실행되고, 오프라인에서도 마지막 데이터가 보입니다.

## 7단계. 친구 공유 & "바탕화면/위젯"처럼 쓰기

- **공유**: URL만 보내면 끝. 각자 6단계로 설치.
- **iPhone/iPad**: 홈 화면 PWA가 위젯 역할. 열 때마다 오늘 요일로 자동 점프.
- **Windows 바탕화면**: 설치된 PWA를 시작 프로그램에 넣으면(앱 설치 후 Edge: `edge://apps` → 앱 우클릭 → 시작 시 실행) 부팅 때마다 시간표가 뜸.
- **iPad를 책상 시계처럼**: Safari 전체화면 + 가이드 접근(Guided Access)으로 상시 표시 가능.

## 운영 메모

- **시트 수정 → 반영**: 폴링 45초라 보통 1분 내 자동 반영. (PC·폰·시트 어디서 고치든 시트만 고치면 모든 기기에 반영)
- **앱 코드 수정 후 배포할 때**: `sw.js` 첫 줄 `const VERSION = 'v1'` 숫자를 올려야(`v2`, `v3`…) 설치된 기기들이 새 셸을 받습니다. 데이터(시트)는 버전과 무관하게 항상 최신.
- **주 추가**: 시트에 같은 형식(헤더행 + 교시 9행)으로 블록을 이어 붙이면 자동 인식됩니다. 주 헤더행의 **C열(월요일) 날짜만 정확하면** 나머지 요일 날짜 오타는 자동 보정됩니다.
- **공유 암호 게이트**(선택): 원하면 간단한 암호 입력 화면을 추가할 수 있음 — 말만 해주세요.
